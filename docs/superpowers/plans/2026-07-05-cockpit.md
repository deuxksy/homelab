- **OS:** Ubuntu 24.04 LTS (Noble)
- **SSH bootstrap:** cockpit VM은 독립 Ubuntu. cloud-init으로 `cockpit-admin` 계정 + SSH 키 생성. (walle의 crong+become 패턴과 별개)
- **Secrets:** 비밀번호/Tailscale auth key는 `proxmox/ansible/secrets.sops.yaml`(Ansible 전용, 신규) — OpenTofu `secrets.sops.yaml`과 분리
- **OpenTofu backend:** `tofu init` 전 `source ~/git/twenty-four-seven-three-sixty-five/.env`로 R2 자격 증명 주입
- **Tailscale Serve 스킴:** `https+insecure://localhost:9090` (Cockpit 자가서명 TLS, CLAUDE.md Gotcha)
- **sudo:** passworded (NOPASSWD 지양)

### Task 1: Ubuntu cloud image template 생성 스크립트

**Files:**

**Interfaces:**
- Produces: walle의 template VM 901 (`ubuntu-2404-tmpl`). 이후 Task 3에서 clone 대상.

Expected: `no existing 901` 또는 기존 template 없음. 있으면 ID 충돌 회피 (902 등으로 변경).

- [ ] **Step 2: 스크립트 작성**

```bash
#!/usr/bin/env bash
# Ubuntu 24.04 LTS cloud image로 Proxmox template 생성
# Usage: ssh crong@walle.bun-bull.ts.net 'bash -s' < scripts/create-ubuntu-template.sh
set -euo pipefail

TEMPLATE_ID="${TEMPLATE_ID:-901}"
NAME="ubuntu-2404-tmpl"
STORAGE="local-lvm"
IMAGE_URL="https://cloud-images.ubuntu.com/noble/current/noble-server-cloudimg-amd64.img"
TMP_IMG="/tmp/ubuntu-2404.qcow2"

echo "==> Downloading Ubuntu 24.04 cloud image"
wget -q -O "$TMP_IMG" "$IMAGE_URL"

# qemu-guest-agent 사전 주입 (agent 기반 IP 인식 선행 조건)
if command -v virt-customize >/dev/null 2>&1; then
  echo "==> Installing qemu-guest-agent into image"
  virt-customize -a "$TMP_IMG" --install qemu-guest-agent || true
fi

echo "==> Creating template VM $TEMPLATE_ID"
qm create "$TEMPLATE_ID" --name "$NAME" --memory 2048 --cores 2 \
  --net0 virtio,bridge=vmbr0 --ostype l26
qm importdisk "$TEMPLATE_ID" "$TMP_IMG" "$STORAGE"
qm set "$TEMPLATE_ID" --scsihw virtio-scsi-pci \
  --scsi0 "${STORAGE}:vm-${TEMPLATE_ID}-disk-0"
qm set "$TEMPLATE_ID" --ide2 "${STORAGE}:cloudinit" \
  --boot c --bootdisk scsi0
qm set "$TEMPLATE_ID" --serial0 socket --vga serial0
qm set "$TEMPLATE_ID" --agent enabled=1

qm template "$TEMPLATE_ID"
rm -f "$TMP_IMG"
echo "==> Template $TEMPLATE_ID ($NAME) 생성 완료"
```

- [ ] **Step 3: 스크립트 실행 (walle)**

- [ ] **Step 4: template 확인**

Run: `ssh crong@walle.bun-bull.ts.net "sudo qm config 901 | grep -E 'template|name|agent'"`
Expected: `template: 1`, `name: ubuntu-2404-tmpl`, `agent: 1` 포함.

- [ ] **Step 2: cockpit.tf 작성**

```hcl
resource "proxmox_virtual_environment_vm" "cockpit" {
  name        = "cockpit"
  vm_id       = var.cockpit_vmid
  node_name   = var.proxmox_node
  description = "Cockpit web management (Ubuntu 24.04 LTS)"
  started     = true
  tags        = ["monitoring", "cockpit"]

  clone {
    vm_id = var.cockpit_template_vmid
    full  = true
  }

  cpu {
    cores = var.cockpit_resources.cores
    type  = "host"
  }

  memory {
    dedicated = var.cockpit_resources.memory
  }

  disk {
    datastore_id = "local-lvm"
    size         = var.cockpit_resources.disk
    interface    = "scsi0"
  }

  agent {
    enabled = true
  }

  initialization {
    user_account {
      username = "cockpit-admin"
      keys     = [file("~/.ssh/id_ed25519.pub")]
    }
    ip_config {
      ipv4 {
        address = "dhcp"
      }
    }
  }

  network_device {
    bridge = "vmbr0"
  }

  vga {}
}
```

> `keys` 경로는 실제 공개키 경로로 조정 (예: `~/.ssh/id_ed25519.pub` 또는 절대경로).

`templates/cockpit-socket-override.conf.j2`:

```ini
[Socket]

ListenStream=
ListenStream=127.0.0.1:{{ cockpit_listen_port }}
```

- [ ] **Step 3: handlers/main.yml 작성**

```yaml
---
- name: Restart cockpit.socket
  ansible.builtin.systemd:
    name: cockpit.socket
    state: restarted

- name: Reload systemd
  ansible.builtin.systemd:
    daemon_reload: true
```

- [ ] **Step 4: tasks/main.yml 작성**

```yaml
---
- name: cockpit 패키지 설치
  ansible.builtin.apt:
    name: cockpit
    state: present
    update_cache: true

- name: cockpit socket override 디렉토리
  ansible.builtin.file:
    path: /etc/systemd/system/cockpit.socket.d
    state: directory
    owner: root
    group: root
    mode: "0755"

- name: cockpit socket loopback override
  ansible.builtin.template:
    src: cockpit-socket-override.conf.j2
    dest: /etc/systemd/system/cockpit.socket.d/override.conf
    owner: root
    group: root
    mode: "0644"
  notify: Reload systemd
  when: cockpit_bind_loopback_only | bool

- name: cockpit.socket 활성화
  ansible.builtin.systemd:
    name: cockpit.socket
    state: started
    enabled: true
    daemon_reload: true

# 방화벽 (cockpit_manage_firewall이 true일 때만)
- name: UFW — 9090 외부 인터페이스 deny (loopback은 systemd socket에서 제한)
  community.general.ufw:
    rule: deny
    port: "{{ cockpit_listen_port }}"
    proto: tcp
  when: cockpit_manage_firewall | bool

- name: UFW — Tailscale 인터페이스만 443 허용 (Serve)
  community.general.ufw:
    rule: allow
    port: "443"
    proto: tcp
    interface: tailscale0
  when:
    - cockpit_manage_firewall | bool
    - cockpit_tailscale_serve | bool

# 하위 task import
- name: 인증 설정 import
  ansible.builtin.import_tasks: auth.yml

- name: Tailscale 조인 import
  ansible.builtin.import_tasks: tailscale_join.yml

- name: Tailscale Serve 구성 import
  ansible.builtin.import_tasks: tailscale_serve.yml
  when: cockpit_tailscale_serve | bool
```
