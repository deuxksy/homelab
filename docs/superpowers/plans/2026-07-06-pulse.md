# Pulse 도입 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **Commit 정책:** 본 레포 CLAUDE.md 규칙상 commit/push는 사용자 명시적 요청 시에만. 각 task의 commit step은 사용자 승인 후 실행.

**Goal:** VM 102(cockpit)에 Pulse Compose(단일 컨테이너)를 배포하고, Tailscale Serve 10000으로 Pulse를 Tailnet에 노출. 동일 playbook은 회사 Ubuntu 서버에 재적용 가능. PatchMon과 동일 VM 통합, Docker Engine 재사용.

**Architecture:** 기존 cockpit role 확장(pulse.yml, pulse-docker-compose.yml.j2 신규). Docker `127.0.0.1:7655` loopback 바인딩. Tailscale Serve 다중 포트(443→Cockpit 기존, 8443→PatchMon 기존, 10000→Pulse 신규). **인증은 Pulse UI setup wizard로 초기화** (env preseed 폐기, B1). sops 변경 없음.

**Tech Stack:** Ansible, Docker CE + Compose v2 (기존), Pulse(rcourtman/pulse:latest, Go/Alpine/SQLite), Tailscale Serve, community.docker 컬렉션 (기존 설치됨).

**Spec:** [docs/superpowers/specs/2026-07-06-pulse-design.md](../specs/2026-07-06-pulse-design.md) (commit d8affee, 2-way 검증 반영)

## Global Constraints

- **배포 대상:** VM 102(cockpit, Ubuntu 24.04 LTS) — 이미 프로비저닝됨, OpenTofu 변경 없음
- **Docker 포트 바인딩:** `"127.0.0.1:{{ cockpit_pulse_internal_port }}:{{ cockpit_pulse_internal_port }}"` loopback only (R6 변수화)
- **Tailscale Serve 포트:** 10000(허용 포트 443/8443/10000 중). 443→Cockpit, 8443→PatchMon 기존 매핑 유지
- **Secrets:** sops 변경 **없음** (B1 — Pulse auth는 UI setup wizard로 초기화, env_file 생성 안 함)
- **인증 env:** `PULSE_AUTH_USER`/`PULSE_AUTH_PASS` 환경변수 **사용 금지** (B1 — Pulse env vars가 UI 설정 override, 평문 잔류)
- **.env 파일:** 생성 안 함 — `environment` 블록에 포트/metrics 설정만 (비민감)
- **healthcheck:** `wget -q --spider http://localhost:7655/api/health` (R3 — `nc -z` 대신 앱 레벨 검증)
- **Ansible group:** `cockpit_hosts`(기존) — VM 102에 그대로 배포. 새 playbook 불필요, 기존 `cockpit.yml` 확장
- **회사 서버 변수:** `cockpit_pulse_enabled` 등 4개 변수로 제어
- **Tailscale Serve 재구성:** reset 기반 4상태 패턴 (R1 — stale 10000 매핑 자동 제거)

`proxmox/ansible/roles/cockpit/templates/pulse-docker-compose.yml.j2`:

```yaml
name: pulse

services:
  pulse:
    image: rcourtman/pulse:{{ cockpit_pulse_version }}
    restart: unless-stopped
    ports:
      - "127.0.0.1:{{ cockpit_pulse_internal_port }}:{{ cockpit_pulse_internal_port }}"
    volumes:
      - pulse_data:/data
    mem_limit: 512m
    cpus: '1.0'
    security_opt:
      - no-new-privileges:true
    environment:
      # B1(Codex): 인증 env vars 사용 안 함 — UI setup wizard로 초기화
      # PULSE_AUTH_USER/PASS를 넣으면 UI 설정 override + 평문 잔류 발생
      FRONTEND_PORT: "{{ cockpit_pulse_internal_port }}"
      PULSE_METRICS_PORT: "9091"
      PULSE_METRICS_BIND_ADDRESS: "127.0.0.1"
      # R4(Codex): PULSE_TELEMETRY 미사용 — 공식 문서 미확인, no-op 가능성
    healthcheck:
      test: ["CMD-SHELL", "wget -q --spider http://localhost:{{ cockpit_pulse_internal_port }}/api/health || exit 1"]
      interval: 30s
      timeout: 5s
      retries: 3
      start_period: 30s
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

volumes:
  pulse_data:
```

- [ ] **Step 1: 현재 tailscale_serve.yml 확인**

Run: `cat -n proxmox/ansible/roles/cockpit/tasks/tailscale_serve.yml`
Expected: 2상태 분기 — `serve_needs_reconfig` set_fact 2개 (Cockpit만 / Cockpit+PatchMon), shell task 2개 (reset + 매핑).

- [ ] **Step 2: tailscale_serve.yml 전체 교체**

`proxmox/ansible/roles/cockpit/tasks/tailscale_serve.yml` 전체를 아래로 교체:

```yaml
---
# Tailscale Serve 다중 포트 매핑 (reset 기반 4상태 동적 패턴, R1/R2)
# 443   → Cockpit (https+insecure, 자가서명 TLS) — 항상
# 8443  → PatchMon (http, Docker loopback) — cockpit_patchmon_enabled 시
# 10000 → Pulse (http, Docker loopback) — cockpit_pulse_enabled 시
# idempotency: 필요 포트 집합 vs 현재 TCP 포트 비교 → 차이 시 reset 후 활성 서비스만 재구성

- name: Tailscale Serve 상태 확인 (JSON, R2)
  ansible.builtin.command: tailscale serve status --json
  register: ts_serve_status
  changed_when: false
  failed_when: false

- name: 현재 Serve 매핑 파싱
  ansible.builtin.set_fact:
    ts_current_serve: "{{ ts_serve_status.stdout | from_json }}"
  when: ts_serve_status.stdout | length > 0

- name: ts_current_serve 기본값 (빈 stdout 대비, R2)
  ansible.builtin.set_fact:
    ts_current_serve: {}
  when: ts_serve_status.stdout | length == 0

# R1: 활성 서비스에 따른 필요 포트 집합 계산
- name: 필요 Serve 포트 집합 계산
  ansible.builtin.set_fact:
    serve_required_ports: >-
      {{
        ['443']
        | union((cockpit_patchmon_enabled | bool) | ternary(['8443'], []))
        | union((cockpit_pulse_enabled | bool) | ternary(['10000'], []))
      }}

# R1: 필요 포트 vs 현재 TCP 포트 비교 → 재구성 필요 여부
# - 필요한 포트가 현재 없거나 (추가 필요)
# - 현재 포트 중 필요 없는 것이 있으면 (stale 제거 필요)
- name: Serve 재구성 필요 여부 판정
  ansible.builtin.set_fact:
    serve_needs_reconfig: >-
      {{
        ts_current_serve.TCP is not defined
        or (serve_required_ports | difference(ts_current_serve.TCP.keys())) | length > 0
        or (ts_current_serve.TCP.keys() | difference(serve_required_ports)) | length > 0
      }}

# reset 후 활성 서비스만 재구성 (stale 매핑 자동 제거, R1)
- name: Tailscale Serve 재구성 (reset 기반)
  ansible.builtin.shell: |
    tailscale serve reset
    tailscale serve --bg --https=443 --set-path / https+insecure://localhost:{{ cockpit_listen_port }}
    {% if cockpit_patchmon_enabled | bool %}
    tailscale serve --bg --https={{ cockpit_patchmon_serve_port }} --set-path / http://localhost:3000
    {% endif %}
    {% if cockpit_pulse_enabled | bool %}
    tailscale serve --bg --https={{ cockpit_pulse_serve_port }} --set-path / http://localhost:{{ cockpit_pulse_internal_port }}
    {% endif %}
  when:
    - serve_needs_reconfig | bool
    - cockpit_tailscale_serve | bool
  changed_when: true
```

> **동적 패턴 핵심:** `serve_required_ports`는 활성 서비스에 따라 `['443']`, `['443','8443']`, `['443','10000']`, `['443','8443','10000']` 중 하나. 현재 `TCP.keys()`와 차이가 있으면 reset 후 재구성. Jinja2 `{% if %}` 조건부로 활성 서비스 매핑만 명령에 포함.

> **PatchMon 호환성:** PatchMon 활성 시 `serve_required_ports`에 `'8443'` 포함, 기존과 동일하게 매핑. Pulse 비활성 시 `'10000'` 제외되어 stale 10000 자동 제거.
