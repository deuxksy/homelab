# PatchMon 도입 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **Commit 정책:** 본 레포 CLAUDE.md 규칙상 commit/push는 사용자 명시적 요청 시에만. 각 task의 commit step은 사용자 승인 후 실행.

**Goal:** VM 102(cockpit)에 Docker Engine + PatchMon Compose(4컨테이너)를 배포하고, Tailscale Serve 8443으로 PatchMon을 Tailnet에 노출. 동일 playbook은 회사 Ubuntu 서버에 재적용 가능.

**Architecture:** 기존 cockpit role 확장(docker.yml, patchmon.yml 신규). Docker `127.0.0.1:3000` loopback 바인딩. Tailscale Serve 다중 포트(443→Cockpit 기존 유지, 8443→PatchMon 신규). 시크릿은 기존 Ansible sops에 patchmon_* 키 추가.

**Tech Stack:** Ansible, Docker CE + Compose v2, PatchMon(ghcr.io/patchmon/patchmon-server), PostgreSQL 17, Redis 7, guacd, Tailscale Serve, sops+age.

**Spec:** [docs/superpowers/specs/2026-07-06-patchmon-design.md](../specs/2026-07-06-patchmon-design.md)

## Global Constraints

- **배포 대상:** VM 102(cockpit, Ubuntu 24.04 LTS) — 이미 프로비저닝됨, OpenTofu 변경 없음
- **Docker 포트 바인딩:** `"127.0.0.1:3000:3000"` loopback only (B3 — LAN 노출 금지)
- **Tailscale Serve 포트:** 8443(허용 포트 443/8443/10000 중, B4). 443→Cockpit 기존 매핑 유지
- **Secrets:** `proxmox/ansible/secrets.sops.yaml`에 patchmon_* 키 5개 추가. playbook pre_tasks sops 패턴(cockpit.yml 준수)
- **.env 권한:** `mode: "0600"`, `owner: root`, `group: root` (R-AGY1)
- **Ansible group:** `cockpit_hosts`(기존) — VM 102에 그대로 배포. 새 playbook 불필요, 기존 `cockpit.yml` 확장
- **회사 서버 변수:** `cockpit_patchmon_enabled`, `cockpit_tailscale_serve` 등 7개 변수로 제어

### Task 3: docker.yml — Docker Engine 설치 태스크

**Files:**
- Create: `proxmox/ansible/roles/cockpit/tasks/docker.yml`

**Interfaces:**
- Consumes: `cockpit_patchmon_enabled`(조건부 import는 main.yml Task 6에서)
- Produces: Docker CE + compose plugin 설치. Task 5(patchmon.yml)에서 `community.docker.docker_compose_v2` 사용 전제.

> heritage.yml의 Docker 설치 패턴(GPG key + apt_repository + apt)을 재사용 가능한 task 파일로 분리. `ansible_distribution_release`로 Ubuntu codename 자동 감지.

- [ ] **Step 1: docker.yml 작성**

`proxmox/ansible/roles/cockpit/tasks/docker.yml`:

```yaml
---
# Docker Engine 설치 (heritage.yml 패턴 재사용)
# PatchMon Compose 배포 전제 조건

- name: Docker GPG 키 디렉토리
  ansible.builtin.file:
    path: /etc/apt/keyrings
    state: directory
    mode: "0755"

- name: Docker GPG 키 다운로드
  ansible.builtin.get_url:
    url: https://download.docker.com/linux/ubuntu/gpg
    dest: /etc/apt/keyrings/docker.asc
    mode: "0644"

- name: Docker apt repository 추가
  ansible.builtin.apt_repository:
    repo: >-
      deb [arch={{ ansible_architecture | replace('x86_64', 'amd64') }}
      signed-by=/etc/apt/keyrings/docker.asc]
      https://download.docker.com/linux/ubuntu
      {{ ansible_distribution_release }} stable
    state: present
    update_cache: true

- name: Docker 패키지 설치
  ansible.builtin.apt:
    name:
      - docker-ce
      - docker-ce-cli
      - containerd.io
      - docker-compose-plugin
    state: present

- name: Docker 서비스 활성화
  ansible.builtin.systemd:
    name: docker
    state: started
    enabled: true
```

> heritage.yml은 Debian 대상이나, Docker repo URL만 `debian`→`ubuntu`로 변경. `ansible_distribution_release`는 Ubuntu 24.04에서 `noble` 반환.

name: patchmon

services:
  server:
    image: ghcr.io/patchmon/patchmon-server:{{ cockpit_patchmon_version }}
    restart: unless-stopped
    env_file: .env
    ports:
      - "127.0.0.1:3000:3000"
    networks:
      - patchmon-internal
    depends_on:
      database:
        condition: service_healthy
      redis:
        condition: service_healthy
      guacd:
        condition: service_healthy
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

  database:
    image: postgres:17-alpine
    restart: unless-stopped
    env_file: .env
    volumes:
      - postgres_data:/var/lib/postgresql/data
    networks:
      - patchmon-internal
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER} -d ${POSTGRES_DB}"]

      timeout: 5s
      retries: 7

      driver: "json-file"

      driver: "json-file"

volumes:
  postgres_data:
  redis_data:

### Task 5: patchmon.yml — Compose 배포 태스크

- [ ] **Step 1: patchmon.yml 작성**

`proxmox/ansible/roles/cockpit/tasks/patchmon.yml`:

```yaml
---
# PatchMon Compose 배포
# 의존성: docker.yml(Docker Engine), playbook patchmon_* facts

- name: PatchMon 배포 디렉토리
  ansible.builtin.file:
    path: "{{ cockpit_patchmon_dir }}"
    state: directory
    owner: root
    group: root
    mode: "0755"

- name: docker-compose.yml 배포
  ansible.builtin.template:
    src: docker-compose.yml.j2
    dest: "{{ cockpit_patchmon_dir }}/docker-compose.yml"
    owner: root
    group: root
    mode: "0644"

- name: .env 배포 (R-AGY1: 0600/root)
  ansible.builtin.template:
    src: patchmon.env.j2
    dest: "{{ cockpit_patchmon_dir }}/.env"
    owner: root
    group: root
    mode: "0600"
  no_log: true

- name: PatchMon Compose 기동 (healthcheck 대기)
  community.docker.docker_compose_v2:
    project_src: "{{ cockpit_patchmon_dir }}"
    state: present
    wait: true
  register: patchmon_compose

- name: PatchMon 컨테이너 상태 확인
  ansible.builtin.assert:
    that:
      - patchmon_compose.containers | length == 4
    fail_msg: "PatchMon 컨테이너 4개(server/database/redis/guacd) 기동 실패"
```

> `wait: true` — `docker compose up --wait` 동등 (healthcheck 통과 대기). `community.docker` 컬렉션 필요 (`ansible-galaxy collection install community.docker`).

> 현재 tailscale_serve.yml은 `tailscale serve reset` + 443 매핑을 수행. reset은 기존 매핑을 모두 지우므로, 8443 추가 후에도 reset 시 8443이 날아가지 않도록 reset 블록에 8443 추가. 또는 reset 없이 개별 매핑 추가 방식 선택. 여기서는 reset 기반 재구성에 8443을 포함.

- [ ] **Step 1: 현재 tailscale_serve.yml 확인**

`proxmox/ansible/roles/cockpit/tasks/tailscale_serve.yml` 전체를 아래로 교체:

```yaml
---
# Tailscale Serve 다중 포트 매핑
# 443 → Cockpit (https+insecure, 자가서명 TLS)
# 8443 → PatchMon (http, Docker loopback)

- name: Tailscale Serve 상태 확인 (JSON, R-AGY2)
  ansible.builtin.command: tailscale serve status --json
  register: ts_serve_status
  changed_when: false

- name: 현재 Serve 매핑 파싱
  ansible.builtin.set_fact:
    ts_current_serve: "{{ ts_serve_status.stdout | from_json }}"
  when: ts_serve_status.stdout | length > 0

- name: Serve 재구성 필요 여부 (Cockpit만)
  ansible.builtin.set_fact:
    serve_needs_reconfig: >-
      {{
        ts_current_serve.Web is not defined
        or ts_current_serve.Web['443'] is not defined
        or (ts_current_serve.Web['8443'] is defined and cockpit_patchmon_enabled | bool == false)
      }}

- name: Serve 재구성 필요 여부 (Cockpit + PatchMon)
  ansible.builtin.set_fact:
    serve_needs_reconfig: >-
      {{
        ts_current_serve.Web is not defined
        or ts_current_serve.Web['443'] is not defined
        or ts_current_serve.Web['8443'] is not defined
      }}
  when: cockpit_patchmon_enabled | bool

- name: Tailscale Serve 구성 (Cockpit만 — PatchMon 비활성화)
  ansible.builtin.shell: |
    tailscale serve reset
    tailscale serve --bg --https=443 --set-path / https+insecure://localhost:{{ cockpit_listen_port }}
  when:
    - serve_needs_reconfig | bool
    - not (cockpit_patchmon_enabled | bool)
  changed_when: true

- name: Tailscale Serve 구성 (Cockpit + PatchMon)
  ansible.builtin.shell: |
    tailscale serve reset
    tailscale serve --bg --https=443 --set-path / https+insecure://localhost:{{ cockpit_listen_port }}
    tailscale serve --bg --https={{ cockpit_patchmon_serve_port }} --set-path / http://localhost:3000
  when:
    - serve_needs_reconfig | bool
    - cockpit_patchmon_enabled | bool
  changed_when: true
```

> `from_json` 파싱으로 `Web["443"]`/`Web["8443"]` 키 존재 여부 확인 (R-AGY2). reset 기반 재구성은 두 매핑을 원자적으로 설정. JSON 스키마는 Tailscale 버전에 따라 상이할 수 있으므로, 실행 시 `ts_current_serve.Web` 구조 확인 필요 — 구조 다를 시 `ts_serve_status.stdout` 문자열 검색 fallback.
